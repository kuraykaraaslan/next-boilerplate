# PLAN/3 — Tooling: Lint Sınırı + Registry Scan + DB Glob

**Bağımlılık:** PLAN/2. **Paralellik:** PLAN/4 ile eşzamanlı. İçindeki 3 alt-iş de birbirine paralel.
**Önce oku:** `PLAN/0.md`, `PLAN/2.md`.

## Amaç
Yeni `server/` vs `ui/` yapısını tooling'e tanıt: framework-bağımsızlık sınırını lint ile zorla, registry tarayıcılarını ve DB entity glob'unu yeni yola güncelle.

## Yapılacaklar
1. **`eslint.config.mjs`** — `no-restricted-imports` ile sınır:
   - `modules/*/server/**` → `react`, `react-dom`, `next`, `@nb/*/ui`, `@nb/*/hooks` YASAK.
   - `modules/*/server/**` → `@nb/*/server` serbest.
   - `ui/` → `server/` import edebilir; `server/` → `ui/` yasak.
2. **`scripts/registry/modules.mjs`**: export bucket glob'ları `modules/<id>/server/**`'e bakacak (eskiden modül kökü).
3. **`scripts/registry/components.mjs`**: `modules/<id>/{ui,hooks}` tarar (eskiden `modules_next/`); **component id formatı stabil kalmalı** (`<id>/ui/<Name>`).
4. **DB datasource**: entity glob `modules/*/server/entities/*.entity.ts`.

## Bitti tanımı (DoD)
- `npm run registry:snapshot` yeni yapıdan doğru tarar (modules/components.json eski veriyle tutarlı).
- eslint `modules/*/server` içinde `react` import'unu reddeder.
- DB entity'leri yüklenir (migration çalışır).

## Dokunulan dosyalar
`eslint.config.mjs`, `scripts/registry/{modules,components}.mjs`, DB datasource config (entity path glob).

## Not
Component id formatını korumak kritik — PLAN/5 codegen import map'i (`<id>/ui/<Name>` → `@nb/<id>/ui/<Name>`) buna bağlı.
