# PLAN/1 — Workspace İskeleti

**Bağımlılık:** yok (ilk). **Paralellik:** solo — herkesi bloklar.
**Önce oku:** `PLAN/0.md`.

## Amaç
npm workspaces zeminini kur; her modülün `@nb/<id>` paketi olabileceği altyapıyı aç. Henüz modül taşıma YOK — sadece iskelet + pilotla doğrulama.

## Yapılacaklar
1. Root `package.json`: `"workspaces": ["modules/*"]`.
2. `next.config.ts`: `transpilePackages`'i **dinamik üret** — config yüklenirken `fs.readdirSync('modules')` ile `@nb/<id>` listesi. (Next, paket TS/TSX kaynağını doğrudan transpile eder; per-paket build YOK.)
3. `tsconfig.json`: `@/*` kalır. `@nb/*` workspace symlink + paket `exports` ile çözülür; editör için gerekirse yardımcı `paths`.
4. Tek **pilot paket** scaffold'u (PLAN/2 pilotuyla aynı modül): minimal `@nb/<pilot>` `package.json` ile `npm install` symlink'ini doğrula.

## Bitti tanımı (DoD)
- `npm install` workspace symlink'lerini kurar (`node_modules/@nb/<pilot>` → `modules/<pilot>`).
- `npm run build` mevcut yapıda kırılmaz.

## Dokunulan dosyalar
`package.json`, `next.config.ts`, `tsconfig.json`, `modules/<pilot>/package.json` (geçici iskelet).

## Notlar / risk
- transpilePackages dinamik üretiminde config senkron `fs` okur (Next config JS); cache'lenir.
- Pilot modül: az-bağımlı, iki-katmanlı bir yaprak (DAG'ın altından). PLAN/2 ile aynı modül olmalı.
