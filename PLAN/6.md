# PLAN/6 — Runtime Registry + Icon Map

**Bağımlılık:** PLAN/5. **Paralellik:** solo (7/8/9'u açar).
**Önce oku:** `PLAN/0.md`, `PLAN/4.md`, `PLAN/5.md`.

## Amaç
Generated `module-runtime.json`'u tüketen, enabled/scope/permission filtreleyen runtime registry'yi yaz; manifest ikon string'lerini `IconDefinition`'a çöz.

## Yapılacaklar
1. **`@nb/common/server/module-registry.ts`**: mevcut `ModuleRegistry` arayüzünü implemente et.
   - **React import ETMEZ** (hem server hem client kullanılabilir). `module-runtime.json` okur.
   - Metotlar `componentId` döner (gerçek komponent client `<Slot>`/widget'ta çözülür).
   - `enabledIds: Set<string>` parametresiyle filtre: `enabledIds.has(moduleId)` + scope eşleşmesi (`'both'` veya istenen) + permission alt kümesi; `order` sonra `id` sırala.
   - `getMenuItems`, `getSlotContributions(slot)`, `getWidgets`, `isModuleEnabled`, `getEnabledModules`.
2. **`@nb/common/ui/icon-map.ts`**: manifest ikon string (`"fas fa-gear"`) → statik `IconDefinition` map (tree-shaking — dinamik FA lookup DEĞİL). Eski `AdminShell.tsx` import bloğundan (~40 ikon) seed. `resolveIcon(name?): IconDefinition | undefined` (`"fas fa-x"` ve bare `"x"` ikisini de çöz).
3. **Href çözümü**: registry ham href döner (`/admin/...`); prefix'i tüketici (AdminShell) `` `/tenant/${tenantId}${href}` `` ekler.

## Bitti tanımı (DoD)
- `module-registry.test.ts` geçer: scope/enabled/permission filtreleme + `order`→id sıralama doğrulanır.

## Dokunulan dosyalar
`@nb/common/server/module-registry.ts`, `@nb/common/ui/icon-map.ts`, `@nb/common/server/module-registry.test.ts`.

## Not
icon-map FA ikonlarını import eder → **client** tarafı. Registry'nin kendisi (server-usable) icon-map'i import ETMEMELİ; ikon çözümü tüketici (AdminShell) tarafında yapılır.
