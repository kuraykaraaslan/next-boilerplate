# PLAN/0 — Orkestrasyon & Ortak Konvansiyonlar (ÖNCE OKU)

> Bu dosya seti, modülleri WordPress-vari eklentilere çevirmenin fazlı planıdır.
> Her `PLAN/n.md` = bir paralel agent'in sahiplenebileceği self-contained iş-paketi.
> Agent kuralı: **`PLAN/0.md` (bu dosya) + kendi dosyanı oku, başka bölüme drift etme.**

## Hedef
Her modül tek `@nb/<id>` workspace paketi olur: kendi admin menüsünü/sayfasını kaydeder, diğer modüllere slot ile komponent enjekte eder, tenant-bazlı runtime aç/kapa olur; ileride app-store'dan kurulabilir.

## Ortak konvansiyonlar (tüm agent'ler buna uyar)
- Scope: **`@nb`** (örn. `@nb/auth`). Paket adı = modül id.
- Modül içi yapı: `server/` (react/next YASAK), `ui/` (react), `hooks/`; kökte `package.json` + `module.json` + `README.md`.
- Subpath exports: `@nb/<id>/server`, `@nb/<id>/ui`, `@nb/<id>/hooks`, `@nb/<id>/module.json`.
- Component id formatı **stabil**: `<id>/ui/<Name>` (örn. `auth/ui/LoginForm`) → import `@nb/<id>/ui/<Name>`.
- Paket yöneticisi: **npm workspaces** (`package-lock.json` mevcut).
- Manifest href'leri tenant-prefix'siz (`/admin/blog`); runtime prefix'ler.
- Bağımlılık grafiği **döngüsüz** (78 manifest doğrulandı) → paket deps güvenli DAG.

## Mevcut zemin (keşif sonucu — bilinmesi gereken)
- Plugin runtime iskeleti **zaten var ama bağlı değil**: `modules/module.schema.json` `menu`/`widgets`/`permissions` tanımlı; `ModuleRegistry` arayüzü var, implementasyon yok; `AdminShell` menüsü 39+ item hardcoded; build-time registry üreteci (`scripts/build-registry-snapshot.mjs` + `scripts/registry/*.mjs`, `prebuild`'de) mevcut.
- `SettingService` (`modules/setting/setting.service.ts`) `getByKeys`/`updateMany` metotları **mevcut** → aç/kapa için reuse.
- `app/tenant/[tenantId]/api/admin-settings/route.ts` → yeni `/api/modules` route'u için şablon.
- eslint (`eslint.config.mjs`) şu an boundary kuralı **içermiyor** — PLAN/3'te eklenecek.
