# PLAN/5 — Build-time Codegen (runtime JSON + component map)

**Bağımlılık:** PLAN/3 + PLAN/4. **Paralellik:** solo.
**Önce oku:** `PLAN/0.md`, `PLAN/3.md`, `PLAN/4.md`.

## Amaç
Manifestlerdeki menü/slot/widget verisini runtime-tüketilebilir JSON'a topla; string component id'lerini Next'in bundle edebileceği `React.lazy` import map'ine çevir (codegen — bu olmadan dinamik komponent çözülemez).

## Yapılacaklar
1. **`scripts/registry/modules.mjs`**: `menu`/`widgets`/`slots`/`settings.tabs`/`permissions`'ı runtime çıktısına taşı (şu an düşürülüyor — sadece temel alanlar tutuluyor).
2. **`scripts/registry/module-runtime.mjs`** (yeni): `buildModuleRuntime(modules, components)` → `{ runtimeJson, componentImports }`.
   - `componentIndex = Map(components → [c.id, {filePath, baseName}])`.
   - Her menu item: order/scope/permission default'la, `moduleId` damgala, `icon`/`href` ham bırak → `runtimeJson.menu[]`.
   - Her widget + slot katkısı: `component` id'sini `componentIndex`'te çöz. **Bulunamazsa build'i hata ile durdur** (typo'yu erken yakala). `componentImports`'a (id ile dedup) ekle; `runtimeJson.{widgets,slots}[]`'a `componentId` ile.
   - `runtimeJson` = `{ generatedAt, menu[], widgets[], slots[], settingsTabs[], permissions[], modules[] }`; deterministik sırala (id).
3. **`scripts/registry/codegen.mjs`** (yeni): `renderComponentMap(componentImports)` →
   ```ts
   // AUTO-GENERATED. DO NOT EDIT.
   import { lazy, type ComponentType } from 'react';
   export const moduleComponents: Record<string, ComponentType<any>> = {
     "payment/ui/BillingProfileTab": lazy(() => import('@nb/payment/ui/BillingProfileTab')
       .then(m => ({ default: m.default ?? m.BillingProfileTab }))),
   };
   ```
4. **`scripts/build-registry-snapshot.mjs`**: `collectComponents()` sonrası `buildModuleRuntime` çağır; iki çıktıyı **kaynak ağacına** yaz (public/ DEĞİL — statik bundle gerek):
   - `@nb/common/generated/module-runtime.json`
   - `@nb/common/generated/module-components.ts`
   Deterministik sırala, commit'le. `prebuild` zaten `registry:snapshot` çalıştırıyor → package.json değişikliği yok.

## Bitti tanımı (DoD)
- `npm run registry:snapshot` iki artifact üretir.
- Geçersiz component id build'i sesli durdurur.
- Çıktı git-deterministik (tekrar çalıştırınca diff yok).

## Dokunulan dosyalar
`scripts/registry/{modules,module-runtime,codegen}.mjs`, `scripts/build-registry-snapshot.mjs`, `@nb/common/generated/{module-runtime.json,module-components.ts}`.

## Risk
`module-components.ts` `lazy`/`import()` içerir → sadece **client** komponentlerden import edilmeli (PLAN/7 `<Slot>`). Server tarafı (PLAN/6 registry) bundan import etmemeli.
