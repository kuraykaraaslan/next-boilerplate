# PLAN/4 — Manifest Şema + Tipler (Plugin Runtime için)

**Bağımlılık:** PLAN/2. **Paralellik:** PLAN/3 ile eşzamanlı. Saf veri/tip işi, kod yürütmez.
**Önce oku:** `PLAN/0.md`.

## Amaç
Slot (komponent enjeksiyon) ve menü-grup kavramlarını manifest şemasına ve TypeScript tiplerine ekle. Runtime kodları bunları PLAN/5+ tüketecek.

## Yapılacaklar
1. **`modules/module.schema.json`**:
   - `properties.slots`: `{ type: "array", items: { $ref: "#/definitions/slotContribution" } }`.
   - `definitions.slotContribution`:
     ```json
     {
       "type": "object",
       "required": ["slot", "component"],
       "properties": {
         "slot":        { "type": "string" },
         "component":   { "type": "string", "description": "component id, örn 'payment/ui/BillingProfileTab'" },
         "order":       { "type": "integer", "default": 100 },
         "scope":       { "type": "string", "enum": ["system","tenant","both"], "default": "both" },
         "permissions": { "type": "array", "items": { "type": "string" } },
         "props":       { "type": "object", "additionalProperties": true }
       }
     }
     ```
   - `definitions.menuItem`'a `group: { type: "string" }` (sidebar bölüm etiketi).
2. **Module tipleri** (ortak tip dosyaları — şu an `modules_next/module.types.ts` + `modules_next/common/module.types.ts`; PLAN/2 sonrası `@nb/common` altına gelir):
   - `SlotContributionJson` (JSON): `{ slot, component, order?, scope?, permissions?, props? }`.
   - `SlotContribution` (runtime): `{ id, slot, componentId, order, scope, permissions, props, moduleId }`.
   - `ModuleJson`'a `slots?: SlotContributionJson[]`; `MenuItemJson`'a `group?: string`.
   - `ModuleRegistry` arayüzüne `getSlotContributions(slot, opts): SlotContribution[]`; `SlotContribution` re-export.

## Bitti tanımı (DoD)
- Şema geçerli JSON Schema; mevcut tüm `module.json`'lar hâlâ valid.
- Tipler `tsc` ile derlenir.

## Dokunulan dosyalar
`modules/module.schema.json`, module tip dosyaları (`module.types.ts` x2 — yeni konumlarında).
