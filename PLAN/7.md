# PLAN/7 — Slot Sistemi (+ host demo)

**Bağımlılık:** PLAN/6. **Paralellik:** PLAN/8, PLAN/9 ile eşzamanlı.
**Önce oku:** `PLAN/0.md`, `PLAN/5.md`, `PLAN/6.md`.

## Amaç
Modüllerin diğer modüllere komponent enjekte etmesini sağla: host'lar `<Slot name="...">` render eder; başka modüller manifestte o slota katkı bildirir. Kullanıcının "dinamik componentler" isteğinin çekirdeği.

## Yapılacaklar
1. **`@nb/common/ui/Slot.tsx`** (`'use client'`):
   - `getSlotContributions(name, { scope, enabledIds })` ile katkıları al.
   - Her katkı: `moduleComponents[componentId]` (PLAN/5 generated map) lazy çözülür.
   - Her komponent ayrı `<Suspense fallback={...}>` + `SlotErrorBoundary` ile sarılır (biri yavaş/bozuk diğerini etkilemez).
   - `order`→id sıralı (registry'den geliyor). Katkı yoksa `children ?? fallback`.
   - Props: `{...c.props}` + passthrough.
2. **`@nb/common/ui/SlotErrorBoundary.tsx`**: küçük class component; bozuk katkı host sayfayı çökertmez, `slot`+`componentId` loglar.
3. **`ModuleEnabledContext` + `useModuleEnabled()`**: `enabledIds`'i context'ten ver (prop-drilling yok). Call-site sadece `<Slot name="user.profile.tabs" />`.
4. **`useSlotContributions(name)`**: veri-tabanlı host'lar (tabs bir array ise) için çözülmüş `[{Comp, props, id}]` döndüren hook.
5. **Demo**: `@nb/payment/module.json`'a `slots: [{ slot: "user.profile.tabs", component: "payment/ui/BillingProfileTab", order: 50 }]`; `@nb/payment/ui/BillingProfileTab.tsx` oluştur; profil tab host'una slot tablarını ekle.

## Bitti tanımı (DoD)
- `payment` açıkken profil sayfasında Billing tabı render edilir.
- `payment` kapatınca tab kaybolur.
- Bozuk contributed komponent `SlotErrorBoundary` ile izole; sayfa ayakta.

## Dokunulan dosyalar
`@nb/common/ui/{Slot,SlotErrorBoundary}.tsx`, `@nb/common/ui/module-enabled.context.tsx` (+ `useModuleEnabled`/`useSlotContributions`), bir profil host komponenti, `@nb/payment/ui/BillingProfileTab.tsx` + `@nb/payment/module.json`.
